## Prompt

You are Claude Code acting as a senior software architect and governance author.

Your task is to CREATE a single, authoritative `constitution.md` file for a hackathon project that evolves across FIVE PHASES (Phase I to Phase V).

Context:
- This project follows Spec-Driven Development using Spec-Kit
- Human developers MUST NOT write manual implementation code
- Claude Code is the sole implementation agent
- The project evolves from a CLI app to a cloud-native, AI-powered distributed system
- This constitution must remain valid for ALL phases without structural rewrite

REQUIREMENTS FOR constitution.md:

1. Declare this document as the SUPREME AUTHORITY over:
   - specs
   - CLAUDE.md
   - source code
   - infrastructure definitions

2. Define IMMUTABLE GLOBAL LAWS that apply to ALL phases, including but not limited to:
   - No manual coding by humans
   - All implementation generated by Claude Code
   - All work must be spec-driven
   - Specs override code
   - Constitution overrides all other documents

3. Define AGENTIC DEVELOPMENT LAW:
   - Specs are written first
   - Plans are generated from specs
   - Tasks are broken down before implementation
   - Claude Code performs implementation
   - Iterative review is mandatory

4. Define PHASE GOVERNANCE (NO implementation details):
   - Phase I: CLI, in-memory data, no persistence, no auth
   - Phase II: Web app, persistent database, REST APIs, authentication
   - Phase III: AI agents, MCP tools, chat interface, stateless services
   - Phase IV: Containers, Kubernetes, Helm, local cluster deployment
   - Phase V: Cloud-native, event-driven architecture, Kafka, Dapr

5. Define SECURITY & IDENTITY LAWS applicable across phases:
   - User identity must come from an authentication system
   - User data isolation is mandatory
   - Stateless authentication is preferred
   - Secrets must never be hard-coded

6. Define AI & AGENT GOVERNANCE (Phase III+):
   - AI agents must not access databases directly
   - All state mutation must occur via MCP tools
   - Conversation state must be persisted
   - Services must remain stateless

7. Define INFRASTRUCTURE NEUTRALITY:
   - Business logic must not depend on deployment environment
   - Containers and Kubernetes are execution platforms, not architecture
   - Cloud provider lock-in must be avoided

8. Define CONFLICT RESOLUTION ORDER:
   Constitution > Specs > CLAUDE.md > Code

9. Define AMENDMENT RULES:
   - Constitution may only be amended between phases
   - Amendments must include justification and changelog

STYLE & OUTPUT RULES:
- Write in clean, professional Markdown
- Use clear section headings
- Do NOT include code snippets
- Do NOT include API schemas, DB schemas, or infra configs
- Tone must reflect architectural authority
- File name MUST be `constitution.md`

Generate ONLY the contents of `constitution.md`.
Do not include explanations, comments, or additional text.
