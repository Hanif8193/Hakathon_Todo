## Todo In-Memory Python Console Application

---

## 1. Phase Identity

**Phase:** I
**Phase Name:** Todo In-Memory Python Console App
**Maturity Level:** Basic
**Governed By:** constitution.md

This specification defines the complete functional and non-functional requirements for Phase I of the project.

---

## 2. Objective

Build a command-line based Todo application that:

- Runs entirely in the console
- Stores all data in memory (no persistence)
- Is implemented exclusively by Claude Code
- Follows Spec-Driven Development using Spec-Kit Plus
- Demonstrates the Agentic Dev Stack workflow

---

## 3. Development Constraints

The following constraints are mandatory:

- No manual code may be written by a human
- All implementation must be generated by Claude Code
- Development must follow:
  1. Specification
  2. Plan generation
  3. Task decomposition
  4. Implementation
  5. Review and iteration
- No features outside this specification are allowed

---

## 4. Functional Requirements

The application MUST support the following five basic features:

### 4.1 Add Todo
- Allow the user to add a new todo item
- Each todo must include:
  - Title (required)
  - Description (optional)
- Each todo must receive a unique identifier

### 4.2 View Todos
- Display a list of all todos
- Each todo must show:
  - ID
  - Title
  - Completion status (complete / incomplete)

### 4.3 Update Todo
- Allow updating:
  - Title
  - Description
- Updates must target a todo by its ID

### 4.4 Delete Todo
- Allow deletion of a todo by ID
- Deleted todos must be removed from memory

### 4.5 Mark Todo Complete / Incomplete
- Allow toggling the completion status of a todo
- Status changes must be immediately reflected in the list view

---

## 5. Non-Functional Requirements

- The application must run entirely in the terminal
- Data must be stored in memory only
- No database, file storage, or external services are allowed
- The application must be deterministic and predictable
- Code must follow clean structure and readability standards

---

## 6. User Interaction Requirements

- The user must interact via command-line prompts
- Clear instructions must be shown for each action
- Errors (such as invalid IDs) must be handled gracefully
- The user must be able to perform multiple actions in one session

---

## 7. Technical Stack

The implementation must use the following technologies:

- Python 3.13 or higher
- UV for environment and dependency management
- Claude Code for all code generation
- Spec-Kit Plus for specification governance

No additional technologies are permitted in Phase I.

---

## 8. Project Structure Requirements

The repository must contain:

- `constitution.md`
- `specs/`
  - `history/` containing all previous specification versions
- `src/` containing Python source code
- `README.md` with setup and usage instructions
- `CLAUDE.md` defining Claude Code operating rules

---

## 9. Out of Scope (Explicitly Forbidden)

The following are NOT allowed in Phase I:

- Databases
- File-based persistence
- Web interfaces
- Authentication or authorization
- AI agents or MCP tools
- Containers or cloud infrastructure

---

## 10. Completion Criteria

Phase I is considered complete when:

- All five basic todo features are implemented
- The application runs successfully in the console
- All constraints defined in this specification are satisfied
- The implementation is fully generated by Claude Code
- The repository structure matches the defined requirements

---

## 11. Amendment Policy

This specification may only be modified:

- Before starting implementation
- Or at the transition to the next phase

All changes must be recorded in `specs/history/`.
